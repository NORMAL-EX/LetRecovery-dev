============================================================
  XP / 2003 i386 文本安装 —— 可选产品密钥
============================================================

装【原版 Windows XP / 2003 的 i386 镜像】（光盘根目录是 \I386，没有
\sources\install.wim 的那种）时，LetRecovery 走的是「硬盘文本安装」：
复制 i386 到目标盘 → 写 NTLDR(setupldr)/txtsetup.sif/winnt.sif → 重启进
蓝底文本安装。

无人值守（winnt.sif）行为：
  • 不放本目录的 productkey.txt：
      用 UnattendMode=DefaultHide —— 跳过 EULA / 区域 / 欢迎，忽略驱动签名，
      管理员空密码 + 首次自动登录；图形阶段【只在「输入产品密钥」那一页停一下】，
      填完即继续，其余全自动。
  • 放了 productkey.txt：
      用 UnattendMode=FullUnattended —— 全程不停顿，真·全自动无人值守。

★ 怎么开启全自动：
  在本目录（bin\xp\）放一个文本文件  productkey.txt ，里面写上你那套 XP/2003
  的产品密钥，形如：
      XXXXX-XXXXX-XXXXX-XXXXX-XXXXX
  规则：
    - 取第一行非注释内容（# 或 ; 开头的行视为注释，忽略）。
    - 密钥要和你所装的【具体版本/渠道】匹配（VLK 量产版 / 零售 / OEM 各不通用，
      不匹配会在文本或图形阶段报「产品密钥无效」）。

注意：
  - 本工具【不内置】任何产品密钥（不分发盗版密钥）。要不要放、放什么由你自己决定。
  - XP/2003 i386 仅支持 Legacy/BIOS + MBR，无法装到 GPT/UEFI 目标。
  - 文本阶段仍会让你确认「安装到哪个分区」（出于安全，不自动选盘，避免抹错盘）。
