# 用户驱动文件夹（按 Windows 版本自动注入）

把驱动放进对应版本的子目录，**安装该版本系统时会自动注入**到目标系统：

| 目录 | 安装哪个系统时注入 | 注入方式 |
|------|--------------------|----------|
| `win7\`  | Windows 7            | DISM 离线注入（`/Add-Driver /Recurse`） |
| `win8\`  | Windows 8 / 8.1      | DISM 离线注入 |
| `win10\` | Windows 10           | DISM 离线注入 |
| `win11\` | Windows 11           | DISM 离线注入 |
| `xp\`    | Windows XP / 2003    | 见下方说明（不是 DISM） |

## 放什么
直接把驱动的 `.inf` / `.sys` / `.cat`（或整个解压后的驱动文件夹）放进去即可，
可以有多层子目录，程序会**递归**查找并注入。文件夹为空则跳过、不影响安装。

## 版本识别
安装时按目标系统的 `\Windows\System32\ntdll.dll` 版本判定：
6.1=win7、6.2/6.3=win8/8.1、10.0 且 build<22000=win10、build≥22000=win11。

## XP 说明
XP/2003 是 NT 5.x，**不支持 DISM 离线注入**。`xp\` 下的
`ahci\` / `nvme\` / `usb3\` 子目录用于**存储控制器驱动**，由内置的 XP 驱动注入
机制处理（注册 boot-start 服务 + CriticalDeviceDatabase，可靠）。通用设备
（网卡/显卡等）的离线注入 XP 不保证生效。

## 适用场景
- **已在 PE 中安装**、或**安装到非系统分区**：直接读取本目录注入。
- **重装当前系统盘**（需重启进 PE）：安装前这些驱动会随配置复制到数据分区，
  重启进 PE 后再注入。
